SON Kiseok Technical Blog

백엔드 개발, Linux 서버 운영, 데이터 분석 시스템화, 보안/감사 전환 과정을 정리하는 기술 블로그입니다.

이 블로그는 단순 학습 기록이 아니라, 실무에서 겪은 문제를 재사용 가능한 기술 문서로 정리하고 이력서와 포트폴리오에서 근거 자료로 활용하기 위한 공간입니다.

이 블로그에서 확인할 수 있는 것

  • Java / Spring Boot 기반 백엔드 개발 경험
  • Linux, Nginx, Apache, Tomcat 기반 웹 서비스 운영 경험
  • PostgreSQL, Oracle, Tibero 등 관계형 데이터베이스 실무 경험
  • 데이터 분석 결과를 웹 서비스와 운영 시스템으로 연결하는 경험
  • 정보보안, 내부통제, 감사/감리 직무 전환을 위한 학습 기록

글 작성 기준

대부분의 글은 아래 구조로 작성합니다.

문제점 → 원인 → 해결 → 실행 방법 → 재발 방지 / 개선 방향 → 포트폴리오 관점의 의미

특정 고객사나 내부 시스템 정보는 공개하지 않고, 실무 경험을 일반화하여 다른 환경에서도 참고할 수 있는 형태로 정리합니다.

카테고리

카테고리는 글의 큰 목적을 나타내고, 태그는 세부 기술과 키워드를 나타냅니다.

  • backend: Java, Spring Boot, API, 인증/권한, 웹 애플리케이션 구조
  • database: SQL, DB 설계, Oracle, PostgreSQL, Tibero, 데이터 모델링
  • infrastructure: Linux, Nginx, Apache, Tomcat, Podman, 배포, 장애 대응
  • data-systemization: 분석 결과를 DB, API, 화면, 운영 시스템으로 연결하는 작업
  • security-audit: 보안, 접근통제, 로그관리, 내부통제, CISA, ISMS-P
  • project-management: PMP, 요구사항 정리, 업무일지, 보고, 고객 커뮤니케이션
  • career: 이직, 포트폴리오, 기술 블로그 운영, 경력 방향

카테고리별 글은 Topics, 자세한 소개는 About, 학습 방향은 Study에서 확인할 수 있습니다.
